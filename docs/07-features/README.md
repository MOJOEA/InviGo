## 1. 🎯 Event Management (การจัดการกิจกรรม)

### สร้างกิจกรรม
**เส้นทาง:** `/events/create`  
**ฟีเจอร์:**
- กรอกชื่อ, รายละเอียด, สถานที่
- ตั้งวัน-เวลาเริ่มและจบ
- กำหนดจำนวนผู้เข้าร่วมสูงสุด
- อัปโหลดรูปภาพหลายรูป (สูงสุด 2MB ต่อรูป)
- รองรับ JPG, PNG, GIF, WebP

**การใช้งาน:**
```
1. คลิก "สร้างกิจกรรม" ในหน้า "กิจกรรมของฉัน"
2. กรอกข้อมูลทั้งหมด
3. อัปโหลดรูปภาพ (ถ้ามี)
4. กด "บันทึก"
```

---

### แก้ไขกิจกรรม
**เส้นทาง:** `/events/{id}/edit`  
**สิทธิ์:** เฉพาะเจ้าของกิจกรรม  
**ฟีเจอร์:**
- แก้ไขข้อมูลทั้งหมด
- เพิ่ม/ลบรูปภาพ
- เปลี่ยนสถานะ (active, cancelled)

---

### ลบกิจกรรม
**เส้นทาง:** POST `/events/{id}/delete`  
**ความปลอดภัย:**
- ยืนยันก่อนลบ (JavaScript confirm)
- ตรวจสอบสิทธิ์เจ้าของ
- ลบข้อมูลที่เกี่ยวข้อง (Cascade)

---

### ดูรายละเอียดกิจกรรม
**เส้นทาง:** `/events/{id}`  
**แสดง:**
- รูปภาพแกลเลอรี่
- ข้อมูลกิจกรรมทั้งหมด
- สถานะ (กำลังจะมาถึง, ปิดรับ, สิ้นสุด)
- จำนวนที่เหลือ
- ปุ่มเข้าร่วม (ถ้าเปิดรับ)

---

## 2. 👥 Registration System (ระบบลงทะเบียน)

### การลงทะเบียน
**เส้นทาง:** `/events/{id}/join`  
**เงื่อนไข:**
- ต้องล็อกอิน
- ไม่ใช่เจ้าของกิจกรรม
- กิจกรรมยังไม่เต็ม
- ยังไม่เคยลงทะเบียน

**ขั้นตอน:**
```
1. ผู้ใช้กด "เข้าร่วม"
2. สร้าง record สถานะ "รออนุมัติ"
3. แสดงในหน้า "การลงทะเบียนของฉัน"
```

---

### การอนุมัติ/ปฏิเสธ
**เส้นทาง:** `/events/{id}/manage`  
**สิทธิ์:** เฉพาะเจ้าของกิจกรรม  
**ตัวเลือก:**
- ✅ อนุมัติ (approved)
- ❌ ปฏิเสธ (rejected)

**ผลลัพธ์:**
- อนุมัติ: ผู้ลงทะเบียนเห็นสถานะเปลี่ยน + สามารถขอ OTP
- ปฏิเสธ: ผู้ลงทะเบียนเห็นสถานะปฏิเสธ

---

### การยกเลิกลงทะเบียน
**เส้นทาง:** `/events/{id}/withdraw`  
**เงื่อนไข:**
- ยังไม่ถูกอนุมัติ (pending) หรือยกเลิกได้ถ้ายังไม่เช็คอิน
- กิจกรรมยังไม่เริ่ม

---

### สถานะการลงทะเบียน

| สถานะ | Badge | คำอธิบาย |
|-------|-------|---------|
| pending | 🟡 เหลือง | รออนุมัติ |
| approved | 🟢 เขียว | อนุมัติแล้ว |
| rejected | 🔴 แดง | ปฏิเสธ |
| checked_in | 🔵 น้ำเงิน | เช็คอินแล้ว |

---

## 3. 🔐 OTP Check-in (ระบบเช็คอิน)

### การสร้าง OTP
**เส้นทาง:** `/events/{id}/otp`  
**สิทธิ์:** ผู้ลงทะเบียนที่ได้รับอนุมัติ  
**ลักษณะ:**
- รหัส 6 หลัก (ตัวเลข)
- อายุ 30 นาที
- ใช้ได้ครั้งเดียว

**หน้าตา:**
```
┌─────────────────────────────┐
│  🔐 รหัส OTP ของคุณ        │
│                             │
│        1 2 3 4 5 6          │
│   (หมดอายุใน 28:45)        │
└─────────────────────────────┘
```

---

### การเช็คอิน (Organizer)
**เส้นทาง:** `/events/{id}/manage` → Check-in  
**ขั้นตอน:**
1. ผู้จัดขอให้ผู้เข้าร่วมแสดง OTP
2. ผู้จัดกรอก OTP ในหน้าจัดการ
3. ระบบตรวจสอบ:
   - OTP ถูกต้อง?
   - ยังไม่หมดอายุ?
   - ยังไม่ถูกใช้?
4. อัปเดตสถานะเป็น "เช็คอินแล้ว"

**ผลลัพธ์:**
- ผู้เข้าร่วม: สถานะเปลี่ยนเป็น "เช็คอินแล้ว"
- ผู้จัด: รายการอัปเดตแบบ real-time (ตามแผน)

---

## 4. 📊 Statistics Dashboard (แดชบอร์ดสถิติ)

### สถิติที่แสดง

#### จำนวนรวม
```
┌─────────┬──────────┬──────────┐
│  ทั้งหมด │ อนุมัติ   │ รออนุมัติ │
│   100   │    80    │    15    │
└─────────┴──────────┴──────────┘
```

#### แยกตามเพศ
```
ชาย: ████████████░░░░ 60%
หญิง: ████████░░░░░░░░ 40%
```

#### แยกตามอายุ
```
18-25: ██████ 20%
26-35: ██████████████ 50%
36+:   ████████ 30%
```

#### สถานะเช็คอิน
```
เช็คอินแล้ว: 45/80 (56%)
ยังไม่เช็คอิน: 35/80 (44%)
```

---

### สถิติส่วนตัว (Profile)
**แสดง:**
- จำนวนกิจกรรมที่สร้าง
- จำนวนการลงทะเบียนทั้งหมด
- จำนวนที่อนุมัติ
- จำนวนที่เช็คอิน

---

## 5. 👤 User Profile (โปรไฟล์ผู้ใช้)

### ข้อมูลที่เก็บ
- อีเมล (ไม่สามารถแก้ไข)
- ชื่อ-นามสกุล
- วันเกิด + อายุคำนวณอัตโนมัติ
- เพศ (icon buttons)
- รูปโปรไฟล์

### การอัปโหลดรูปโปรไฟล์
**ขนาด:** สูงสุด 2MB  
**รูปแบบ:** JPG, PNG  
**การแสดง:** วงกลม (rounded-full)

**UX:**
- คลิกที่รูปเพื่อเปลี่ยน
- Preview ก่อนบันทึก
- Fallback เป็น DiceBear avatar

---

## 6. 🔔 Notifications (การแจ้งเตือน)

### Flash Messages
ระบบข้อความแจ้งเตือนแบบชั่วคราว

**ประเภท:**
- ✅ Success (เขียว): สำเร็จ
- ❌ Error (แดง): ผิดพลาด
- ⚠️ Warning (เหลือง): เตือน
- ℹ️ Info (ฟ้า): ข้อมูล

**ตัวอย่าง:**
```
┌──────────────────────────────────┐
│  ✅ ลงทะเบียนสำเร็จ!              │
│     กรุณารอการอนุมัติจากผู้จัด   │
└──────────────────────────────────┘
```

## 7. 🎨 UI/UX Features (ฟีเจอร์การแสดงผล)

### Responsive Design
- **Desktop:** Sidebar navigation
- **Mobile:** Bottom tab navigation
- **Tablet:** ปรับตามขนาด

---

### Neo-brutalism Design
**ลักษณะ:**
- กรอบดำหนา (border-2 border-black)
- เงาดำ (box-shadow)
- สีสันสดใส
- ตัวอักษรหนา (font-black)

**คอมโพเนนต์:**
- `neo-box`: กล่องมีเงา
- `neo-card`: การ์ดมี hover
- `neo-btn`: ปุ่มมีเงา
- `neo-input`: input มีกรอบ

---

### Guest Access
**ฟีเจอร์:**
- ดูกิจกรรมได้โดยไม่ต้องล็อกอิน
- เมนูที่ใช้ไม่ได้แสดงสีเทา
- กดไม่ได้ + tooltip "กรุณาเข้าสู่ระบบ"
- ปุ่ม login ในหน้า profile

---

### Iconography
**ชุดไอคอน:** Material Symbols Outlined  
**ขนาด:** text-xl สำหรับ mobile, default สำหรับ desktop

**ไอคอนที่ใช้:**
- search (ค้นหา)
- calendar_month (กิจกรรม)
- confirmation_number (ลงทะเบียน)
- person (โปรไฟล์)
- add (สร้าง)
- logout (ออกจากระบบ)

---

### Loading States
**ที่มี:**
- ปุ่ม submit มี loading state
- (แผน) Skeleton loading สำหรับ content
